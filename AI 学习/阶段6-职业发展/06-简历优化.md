## STAR 法则

### 结构说明

```
S - Situation（情境）
    项目背景、面临的挑战

T - Task（任务）
    你的具体职责、目标

A - Action（行动）
    你采取的具体措施、技术方案

R - Result（结果）
    量化的成果、业务影响
```

### 项目描述模板

```markdown
## [项目名称] | [时间]

**背景**：[公司/团队面临什么问题，为什么需要这个项目]

**职责**：[你在项目中的角色和主要负责内容]

**技术方案**：
- 使用 [技术1] 实现 [功能1]
- 使用 [技术2] 解决 [问题1]
- 采用 [技术3] 优化 [指标1]

**成果**：
- [量化指标1]：从 X 提升到 Y（+Z%）
- [量化指标2]：XX
- [业务影响]：XX
```

## 项目描述示例

### 示例 1：RAG 知识库项目

```markdown
## 企业知识库智能问答系统 | 2024.03 - 2024.06

**背景**：公司内部文档分散在多个系统，员工查找信息平均耗时 15 分钟，
急需一个智能化的知识检索系统。

**职责**：独立负责系统的技术选型、架构设计和核心开发。

**技术方案**：
- 基于 LangChain + ChromaDB 构建 RAG 流水线
- 使用 BGE-large 进行文档向量化，支持中英文混合检索
- 采用 Hybrid Search (BM25 + 向量) + Rerank 优化检索效果
- 设计 Multi-Query 策略，提升长尾问题覆盖率
- 使用 FastAPI 提供 REST API，支持流式响应

**成果**：
- 检索准确率：92%（Recall@5），超过基线 15%
- 平均响应时间：1.5s（P99 < 3s）
- 支持 10 万+ 文档，日均问答 2000+ 次
- 员工信息查找时间从 15 分钟降至 30 秒
```

### 示例 2：Agent 项目

```markdown
## 多 Agent 自动化运营系统 | 2024.06 - 2024.09

**背景**：运营团队每天需要手动收集数据、生成报告、发送邮件，
耗时 3-4 小时，希望用 AI 自动化这些流程。

**职责**：负责 Agent 系统架构设计和核心 Agent 开发。

**技术方案**：
- 基于 LangGraph 设计多 Agent 协作架构
- 实现数据采集 Agent（API 对接）、分析 Agent（数据处理）、
  报告 Agent（内容生成）三个专业 Agent
- 设计 Supervisor Agent 进行任务分配和结果整合
- 集成 Slack/邮件通知，支持人工审核干预

**成果**：
- 自动化完成率：95%（仅 5% 需人工干预）
- 运营效率提升：从 4 小时降至 30 分钟
- 报告质量评分：4.5/5（运营团队评估）
- 月节省人力成本约 2 万元
```

### 示例 3：模型微调项目

```markdown
## 金融领域大模型微调 | 2024.04 - 2024.05

**背景**：通用大模型在金融专业问答中表现不佳，
需要微调一个领域模型用于投资研究助手。

**职责**：负责数据集构建、模型微调和评估。

**技术方案**：
- 构建 5000+ 条金融 QA 数据集（年报、研报、公告）
- 基于 Qwen2-7B 使用 QLoRA (rank=64) 进行微调
- 使用 LLaMA-Factory 框架，A100 训练 3 小时
- 设计金融专业评估集，从多维度评估模型能力

**成果**：
- 金融问答准确率：从 62% 提升至 85%（+23%）
- 研报摘要质量：人工评分从 3.2 提升至 4.3
- 推理速度：与基座模型持平（无性能损失）
- 模型已部署服务 20+ 分析师日常使用
```
