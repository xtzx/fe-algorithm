### 技能提升矩阵

```
              广度 →
            低      中      高
      ┌─────────────────────────┐
   低 │ 初学者  │ 多面手  │ 杂而不精│
深    │         │         │        │
度 中 │ 专才    │ T型人才 │ π型人才│
 ↓    │         │  (目标) │ (高级) │
   高 │ 偏科    │ 专家    │ 全能    │
      └─────────────────────────┘

目标：先深后广，T型 → π型
- 先在一个方向做深（如 RAG）
- 再扩展相关领域（如 Agent、微调）
```

## 转型案例参考

### 案例：后端工程师 → LLM 应用工程师

```
背景：
- 3 年 Java 后端经验
- 想转型 AI 方向
- 0 基础开始

第 1-2 个月：基础学习
├── 学习 Python（已有编程基础，快速上手）
├── 了解 AI/ML 基础概念
├── 跑通 OpenAI API 示例
└── 产出：能用 API 做简单应用

第 3-4 个月：RAG 入门
├── 学习 LangChain 基础
├── 搭建第一个 RAG Demo
├── 理解向量检索原理
└── 产出：个人知识库项目

第 5-6 个月：深入优化
├── 学习检索优化技术
├── 项目迭代（Rerank、Hybrid）
├── 开始写技术博客
└── 产出：完整的 RAG 项目 + 2 篇博客

第 7-8 个月：扩展技能
├── 学习 Agent 开发
├── 了解微调基础
├── 参与开源项目
└── 产出：Agent 项目 + 开源贡献

第 9-10 个月：求职准备
├── 完善作品集
├── 优化简历
├── 面试准备
└── 开始投递

第 11-12 个月：入职
├── 拿到 Offer
├── 成功转型
└── 继续学习成长

关键成功因素：
1. 利用已有优势（后端经验、工程能力）
2. 聚焦一个方向（RAG）做深
3. 持续输出（博客、项目）
4. 主动参与社区
```
