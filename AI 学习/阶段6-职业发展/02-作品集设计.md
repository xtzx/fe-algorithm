# 💼 02 - 作品集设计

> 构建有竞争力的项目组合

---

## 作品集组合策略

### 理想组合结构

```
作品集 = 2 旗舰项目 + 1-2 技术亮点项目

旗舰项目（Flagship）：
- 完整的业务闭环
- 体现综合能力
- 可以演示和讲解
- 有量化成果

技术亮点项目（Highlight）：
- 深入某个技术点
- 体现技术深度
- 有性能/指标对比
- 代码质量高
```

### 项目选择原则

```
1. 可演示性
   - 有可运行的 Demo
   - 效果直观可见
   - 能在面试中演示

2. 可量化性
   - 有性能指标
   - 有对比数据
   - 有改进幅度

3. 技术代表性
   - 覆盖核心技能
   - 体现技术选型
   - 展示最佳实践

4. 差异化
   - 不要太常见的项目
   - 有个人创新点
   - 解决实际问题
```

## 旗舰项目推荐

### 项目 A：企业知识库 RAG 助手

```
项目定位：展示 RAG 全栈能力

功能范围：
MVP 版本：
- PDF/文档上传和解析
- 文档切分和向量化
- 语义检索 + 问答
- 基础 Web 界面

完整版本：
- 多格式支持（PDF/Word/网页）
- 多轮对话
- 来源引用展示
- 用户反馈收集

优化版本：
- Rerank 精排
- Hybrid Search
- 流式输出
- 性能优化

技术栈推荐：
- 后端：FastAPI + LangChain
- 向量库：ChromaDB / Milvus
- LLM：OpenAI API / 本地 Qwen
- 前端：Streamlit / React
- 部署：Docker + Nginx

量化指标：
- 检索准确率 (Recall@5)：X%
- 回答准确率：X%
- 平均响应时间：X ms
- 支持文档数：X 万
- QPS：X

里程碑：
Week 1-2：MVP，基础 RAG 功能
Week 3-4：完整版，多格式 + 对话
Week 5-6：优化版，Rerank + 部署
```

```
项目定位：展示 Agent 设计和编排能力

量化指标：
- 任务完成率：X%
- 平均执行时间：X 分钟

```

```
项目定位：展示模型微调全流程能力

产出物：
```

## 项目开发流程

### 标准开发流程

```
Phase 1：规划（1 周）
├── 明确项目目标和范围
├── 技术选型
├── 里程碑拆分
└── 创建 GitHub 仓库

Phase 2：MVP（2-3 周）
├── 核心功能开发
├── 基础测试
├── 初步文档
└── 可演示版本

Phase 3：完善（2-3 周）
├── 功能完善
├── 性能优化
├── 完整文档
└── 部署上线

Phase 4：包装（1 周）
├── README 完善
├── Demo 视频
├── 技术博客
└── 量化指标总结
```
