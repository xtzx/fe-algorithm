## 1. AI 行业发展趋势

### 1.1 2024-2025 热门方向

```
🔥 高热度方向：

1. LLM 应用开发
   └── RAG、Agent、Prompt Engineering

2. AI Infra / MLOps
   └── 推理优化、模型部署、平台建设

3. 多模态 AI
   └── 图像理解、视频生成、语音交互

4. AI Agent
   └── 自主决策、工具调用、多 Agent 协作

5. AI 安全
   └── 对齐、红队测试、内容安全
```

### 1.2 行业机会分析

```
高增长领域：
├── 企业 AI 应用（效率提升、智能客服）
├── 垂直行业解决方案（医疗、金融、法律）
├── 开发者工具（AI 编程助手、代码生成）
├── 内容创作（文本、图像、视频生成）
└── AI 基础设施（推理服务、向量数据库）

相对成熟领域：
├── 推荐系统
├── 搜索优化
├── 传统 CV/NLP
└── 数据分析/BI
```

### 1.3 技术趋势

```
正在兴起：
✅ Agentic Workflow（多步骤 AI 工作流）
✅ RAG 2.0（GraphRAG、Agentic RAG）
✅ 小模型崛起（Phi、Qwen 等高效模型）
✅ 边缘 AI（端侧推理）
✅ AI 原生应用（非套壳产品）

持续重要：
✅ Prompt Engineering
✅ Fine-tuning（LoRA、QLoRA）
✅ Vector Database（Embedding 检索）
✅ LLM 推理优化

逐渐弱化：
⬇️ 传统 NLP（规则、传统模型）
⬇️ 小规模预训练
```

## 2. 岗位类型与要求

### 2.1 岗位分类

```
LLM/AI 相关岗位分类：

1. LLM 应用工程师 / AI 应用开发
   - 核心：Prompt、RAG、Agent 开发
   - 技能：Python、LangChain、向量数据库
   - 定位：产品落地导向

2. AI 算法工程师
   - 核心：模型训练、微调、优化
   - 技能：PyTorch、Transformer、分布式训练
   - 定位：模型能力提升

3. AI Infra / MLOps 工程师
   - 核心：模型部署、推理优化、平台建设
   - 技能：vLLM、Docker、K8s、监控
   - 定位：工程效率和稳定性

4. AI 产品经理
   - 核心：AI 产品设计、场景挖掘
   - 技能：产品思维、AI 理解、数据分析
   - 定位：商业价值落地

5. 提示词工程师 / Prompt Engineer
   - 核心：Prompt 优化、评估
   - 技能：Prompt 技巧、评估方法
   - 定位：AI 能力调优
```

### 2.3 JD 分析示例

```
【LLM 应用工程师 JD 示例】

岗位职责：
1. 负责 LLM 应用的设计和开发
2. 构建和优化 RAG 系统
3. 开发 AI Agent 和工作流
4. 与产品/业务团队协作落地 AI 能力

任职要求：
1. 计算机相关专业，3年+ 开发经验
2. 熟悉 Python，了解 Web 开发
3. 熟悉 LangChain/LlamaIndex 等框架
4. 了解向量数据库（Milvus/Qdrant/ChromaDB）
5. 了解 Prompt Engineering 和 RAG
6. 加分项：有 Agent 开发经验

翻译：
- 核心能力：RAG + Agent + Prompt
- 技术栈：Python + LangChain + 向量库
- 软技能：与业务协作、产品落地
```

## 3. 薪资水平参考

### 3.1 国内薪资范围（2024）

```
一线城市（北上广深杭）：

LLM 应用工程师：
├── 初级（0-2年）：20-35K/月
├── 中级（2-5年）：35-50K/月
└── 高级（5年+）：50-80K/月

AI 算法工程师：
├── 初级（0-2年）：25-40K/月
├── 中级（2-5年）：40-60K/月
└── 高级（5年+）：60-100K/月

AI Infra 工程师：
├── 初级（0-2年）：25-40K/月
├── 中级（2-5年）：40-60K/月
└── 高级（5年+）：60-90K/月

注：以上为税前月薪，不含年终奖和股票
```

### 3.2 影响薪资的因素

```
薪资影响因素：

1. 公司类型
   - 大厂（BAT、字节等）：高薪 + 股票
   - AI 独角兽：高薪 + 期权
   - 传统企业：相对稳定
   - 创业公司：变动大

2. 技术深度
   - 有落地项目经验 > 只有学习经验
   - 有开源贡献 > 纯闭源开发
   - 有量化指标 > 只有定性描述

3. 业务匹配度
   - 垂直领域经验加分
   - 相关行业背景加分

4. 面试表现
   - 项目讲解清晰
   - 技术深度足够
   - 软技能良好
```

## 5. 求职策略

### 5.1 不同背景的转型建议

```
后端工程师 → LLM 应用：
├── 优势：工程能力强、熟悉服务化
├── 补充：LLM 基础、RAG/Agent 实战
├── 路径：从公司内部 AI 项目切入
└── 定位：LLM 应用工程师 / AI Infra

前端工程师 → AI 产品：
├── 优势：用户体验、产品理解
├── 补充：AI 基础知识、Prompt 技巧
├── 路径：参与 AI 产品前端开发
└── 定位：AI 产品工程师 / AI PM

数据分析师 → AI 算法：
├── 优势：数据处理、统计基础
├── 补充：深度学习、模型训练
├── 路径：从数据相关的 AI 项目切入
└── 定位：LLM 应用 / 轻量算法

算法工程师 → LLM 算法：
├── 优势：模型基础强
├── 补充：LLM 特定技术、大模型调优
├── 路径：直接转型或内部转岗
└── 定位：LLM 算法 / AI 研究
```

```
求职准备时间线：

```
