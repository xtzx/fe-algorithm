## 2. 本项目可视化使用说明

### 2.1 运行方式

```bash
# 进入可视化目录
cd algorithm/排序/可视化

# 方式 1: 直接用浏览器打开
open index.html  # macOS
# 或
start index.html  # Windows

# 方式 2: 使用本地服务器（推荐，避免 ES Module 问题）
npx serve .
# 然后访问 http://localhost:3000
```

### 2.2 支持的算法

| 算法 | 文件 | 特点说明 |
|------|------|---------|
| 冒泡排序 | `bubble.ts` | 相邻比较交换，看"冒泡"过程 |
| 快速排序 | `quick.ts` | 分区过程，pivot 选择 |
| 归并排序 | `merge.ts` | 分治合并，观察递归树 |
| 堆排序 | `heap.ts` | 堆化过程，观察堆结构 |

## 3. 可视化学习建议

### 3.1 学习路径

```mermaid
flowchart LR
    A[冒泡排序] --> B[选择排序]
    B --> C[插入排序]
    C --> D[归并排序]
    D --> E[快速排序]
    E --> F[堆排序]

    style A fill:#c8e6c9
    style D fill:#bbdefb
    style E fill:#bbdefb
    style F fill:#bbdefb
```

### 3.2 观察要点

#### 冒泡排序

```
观察:
- 每一轮，最大的元素"冒泡"到末尾
- 绿色区域（已排序）从右向左扩展
- 相邻元素比较，看黄色高亮

思考:
- 为什么最坏情况是逆序？
- 为什么近乎有序时很快？（提前终止优化）
```

#### 快速排序

```
观察:
- pivot 的选择（紫色）
- 分区过程：小于 pivot 的去左边，大于的去右边
- 递归深度

思考:
- 什么情况下分区不均匀？
- 随机 pivot vs 固定 pivot 的区别
```

#### 归并排序

```
观察:
- 分解阶段：数组不断二分
- 合并阶段：有序子数组两两合并
- 辅助数组的使用

思考:
- 为什么需要额外空间？
- 为什么是稳定的？
```

#### 堆排序

```
观察:
- 建堆过程：从无序到大顶堆
- 排序过程：堆顶与末尾交换，重新堆化
- 堆的结构变化

思考:
- 为什么建堆是 O(n)？
- 为什么不稳定？
```

### 3.3 推荐练习

1. **对比观察**：用 Sorting.at 同时观察冒泡、选择、插入在相同数据下的表现
2. **边界测试**：用不同数据分布（随机、有序、逆序）观察同一算法
3. **步进调试**：用 VisuAlgo 的步进功能，配合代码理解每一步
4. **计数观察**：关注比较次数和交换次数，验证复杂度分析
